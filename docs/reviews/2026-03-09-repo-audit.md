# ADHD-CADDI Repo Audit

Date: 2026-03-09

## Summary

- Code Health Score: 72/100
- Release Readiness Score: 6.7/10

These are manual audit scores based on the current repository state, recent recovery work, CI behavior, and a targeted scan of risky areas. They are not generated by `desloppify`, which is not installed in this environment.

## How To Read These Scores

- `72/100` means the codebase is now in decent working shape, but it still has enough fragile areas that maintenance will stay expensive unless those areas are tightened.
- `6.7/10` means the product is close to being shippable on web, but Android release verification is still blocking a clean "ready to ship" call.

## What Improved

- Core web navigation is now truthful and test-covered for previously broken routes.
- GitHub Pages deployment now validates the live site instead of only validating a local build.
- Web OAuth storage no longer persists access tokens in browser storage.
- Bubble and tutorial behavior now matches the intended product contract and is covered by corrected browser tests.

## Main Risks Still Present

1. Android release verification is still failing after the APK is built.
   The problem is now in emulator install/launch reliability, not in compilation.

2. Some older end-to-end tests still appear to model pre-hardening OAuth behavior.
   Several test files still write OAuth-related data directly into `localStorage`, which makes them risky as long-term truth sources.

3. There are still "escape hatch" types and lint suppressions in diagnostic and style-heavy areas.
   These do not necessarily break runtime behavior, but they make future breakage easier to hide.

4. Offline behavior is still partially aspirational.
   The docs are now more truthful, but the repo still contains offline-oriented tests and UI paths that deserve one dedicated verification pass.

## Recommended Next Fix Order

1. Finish Android release launch verification.
2. Rewrite or retire stale OAuth/browser tests so they match the new security contract.
3. Reduce `any` casts and broad lint suppressions in diagnostics and styling hotspots.
4. Do one explicit offline/PWA truth pass and either fully enable it or keep it clearly disabled everywhere.

## Evidence Notes

- Web CI and Pages validation are green after the recovery work.
- Android debug CI is green.
- Android release APK assembly is green.
- Android release launch verification is the active blocker.
